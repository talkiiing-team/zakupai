### Документация к модулю объединения всех данных для распределения и извлечения необходимых признаков (Модуль представления результатов распределения в необходиомом формате.py)

Этот модуль реализует алгоритм предобработки данных, включающий в себя:
удаление пропусков, фильтрацию основных средств по дате, объединение зданий и их площадей,
получение кодов услуг по их номеру и т.д.

Т.к. предобработка данных занимает существенное время, код распараллелен на 5 потов и работает
мультипроцессорно.

Основной функционал реализован в классе **ContractsMerger**, принимающий таблицы с данными
и производящий их объединение посредством метода **start_merging**.

### 1. Инициализация
Класс ContractsMerging принимает таблицы, необходимые для дальнейшего объединения и прдеобработки,
в формате pd.DataFrame.
Список таблиц на вход:
- pays_df, счета на оплату (сконкатенированные в один DF);
- merger_df, связь договор - здание;
- main_costs_df, основные средства;
- squares, площади зданий;
- serv_codes, коды услуг.

### 2. Обработка одного договора
Внутренний метод для объединения одной строчки из pays_df. Используется внутри класса для распараллеливания.
Принимает на вход одну строчку из pays_df, возвращает список строк с данными о каждом
распределенном средстве, для которого будет предсказываться стоимость.

### 3. Объединение всей выборки
Метод для получения распределенных средств для счетов на оплату из pays_df. Ничего не принимает на вход,
работает с данными, переданными при инициализации. В среднем занимает до двух минут.
Работает в 5 процессов, можно увеличить в коде в зависимости от ресурсов системы.
Возвращает pd.DataFrame таблицу со всеми распределенными счетами на оплату и список признаков,
используемых в графе для подсчёта стоимости распределенных счетов.
**Важно**:
система может поддерживать добавление новых признаков для зданий и основных средств.
Если в какой-либо из таблиц добавляется новых признак (например, в main_costs_df для каждого основного средства),
то алгоритм сам поймёт его тип (numerical, bool, datetime) и сохранит в список признаков.

### Пример использования
```python
import pandas as pd
from merge_contracts import ContractsMerger

pays_df1 = pd.read_excel("data/Счета на оплату 3800-2023.XLSX")
pays_df2 = pd.read_excel("data/Счета на оплату n.XLSX")
pays_df = pd.concat([pays_df1, pays_df2], axis=0)

merger_df = pd.read_excel("data/Связь договор - здания.XLSX")
main_costs_df = pd.read_excel("data/Основные средства.XLSX")
squares = pd.read_excel("data/Площади зданий.XLSX")
serv_codes = pd.read_excel("data/Коды услуг.XLSX")

merger = ContractsMerger(pays_df, merger_df, main_costs_df, squares, serv_codes)
res, cols = merger.start_merging()
```
